## 类的特性和概念分类

### 接口描述

获取类有的特性，和概念分类 (is a, belong to)


### 接口详情

| 地址     | http://106.14.239.166/contest/api/api_knowledge/ |
| -------- | ------------------------------------------ |
| 请求方式 | POST                                       |

### 请求参数

| 参数             | 说明           |
| ---------------- | -------------- |
| `qualified_name` | 类的全限定名称 |

### 返回参数

| 返回参数                | 说明                                       |
| ----------------------- | ------------------------------------------ |
| `relation` | 关系名称（对于category来说有：is a，part of，consist of）对于特性来说，has characteristic                       |
| `characteristic`      | 特性                 |
| `category`      | 概念分类                 |
| `message`  | 提示无法根据名字找到节点 |

### 调取示例

```json
请求示例:
{
    "qualified_name":"org.jabref.gui.actions.OldDatabaseCommandWrapper"
}

返回示例:
{
    "category": [
        {
            "id": 41169,
            "name": "command",
            "relation": "is a"
        },
        {
            "id": 40598,
            "name": "database",
            "relation": "part of"
        },
        {
            "id": 40572,
            "name": "command",
            "relation": "part of"
        },
        {
            "id": 40599,
            "name": "old database comm",
            "relation": "consist of"
        },
        {
            "id": 40596,
            "name": "wrapper",
            "relation": "consist of"
        }
    ],
    "characteristic": [
        {
            "id": 41170,
            "name": "executable if a database is open",
            "relation": "has characteristic"
        }
    ],
    "message": ""
}
```



## dc文档信息接口

### 接口描述

获取dc文档信息内容

### 接口详情

| 地址     | http://106.14.239.166/contest/api/get_doc/ |
| -------- | ------------------------------------------ |
| 请求方式 | POST                                       |

### 请求参数

| 参数             | 说明           |
| ---------------- | -------------- |
| `qualified_name` | 类的全限定名称 |

### 返回参数

| 返回参数                | 说明                                       |
| ----------------------- | ------------------------------------------ |
| `full_html_description` | 全部html文档描述信息                       |
| `full_description`      | 除去html标签的文档描述信息                 |
| `sentence_description`  | 对full_description进行分句后的文档描述信息 |

### 调取示例

```json
请求示例:
{
    "qualified_name": "org.jabref.model.entry.BibEntry"
}

返回示例:
{
    "full_description": "Represents a BibTex / BibLaTeX entry. In case you search for a builder as described in Item 2 of the book \"Effective Java\", you won't find one. Please use the methods withCiteKey(String) and String).",
    "full_html_description": "Represents a BibTex / BibLaTeX entry.\r\n\r\nIn case you search for a builder as described in Item 2 of the book \"Effective Java\", you won't find one. Please use the methods {@link #withCiteKey(String)} and {@link #withField(Field, String)}.",
    "sentence_description": [
        "Represents a BibTex / BibLaTeX entry.",
        "In case you search for a builder as described in Item 2 of the book \"Effective Java\", you won't find one.",
        "Please use the methods withCiteKey(String) and String)."
    ]
}
```



## 关键方法接口

### 接口描述

返回类下面pagerank值最高的5个方法名

### 接口详情

| 地址     | http://106.14.239.166/contest/api/key_methods/ |
| -------- | ---------------------------------------------- |
| 请求方式 | POST                                           |

### 请求参数

| 参数             | 说明           |
| ---------------- | -------------- |
| `qualified_name` | 类的全限定名称 |

### 返回参数

| 返回参数      | 说明                              |
| ------------- | --------------------------------- |
| `key_methods` | 以列表形式返回类下面5个最相关方法 |

### 调取示例

```json
请求示例:
{
    "qualified_name": "org.jabref.model.entry.BibEntry"
}

返回示例:
{
    "key_methods": [
        "org.jabref.model.entry.BibEntry.getSourceField(Field targetField,EntryType targetEntry,EntryType sourceEntry)",
        "org.jabref.model.entry.BibEntry.replaceKeywords(KeywordList keywordsToReplace,Keyword newValue,Character keywordDelimiter)",
        "org.jabref.model.entry.BibEntry.setField(Field field,String value,EntriesEventSource eventSource)",
        "org.jabref.model.entry.BibEntry.getEntryLinkList(Field field,BibDatabase database)",
        "org.jabref.model.entry.BibEntry.setEntryLinkList(Field field,List<ParsedEntryLink> list)"
    ]
}
```


## 样例代码接口

### 接口描述

返回全限定名称所对应的样例代码

### 接口详情

| 地址     | http://106.14.239.166/contest/api/sample_code/ |
| -------- | ---------------------------------------------- |
| 请求方式 | POST                                           |

### 请求参数

| 参数             | 说明                 |
| ---------------- | -------------------- |
| `qualified_name` | 全限定名称(类或方法) |

### 返回参数

| 返回参数      | 说明                               |
| ------------- | ---------------------------------- |
| `key_methods` | 以列表形式返回5个最短的sample code |

### 调取示例

```json
请求示例:
{
    "qualified_name": "org.jabref.model.entry.BibEntry"
}

查找成功返回示例:
{
    "sample_code": [
        "\n\n    @BeforeEach\n    public void setUp() {\n        action = new MergeReviewIntoCommentMigration();\n    }\n",
        "\n\n    @Override\n    public boolean isActionNecessary(ParserResult parserResult) {\n        return MergeReviewIntoCommentMigration.needsMigration(parserResult);\n    }\n",
        "\n\n    public static List<BibEntry> collectConflicts(ParserResult parserResult) {\n        List<BibEntry> entries = Objects.requireNonNull(parserResult).getDatabase().getEntries();\n        return entries.stream().filter(MergeReviewIntoCommentMigration::hasReviewField).filter(MergeReviewIntoCommentMigration::hasCommentField).collect(Collectors.toList());\n    }\n",
        "\n\n    public void performMigration(ParserResult parserResult) {\n        List<BibEntry> entries = Objects.requireNonNull(parserResult).getDatabase().getEntries();\n        entries.stream().filter(MergeReviewIntoCommentMigration::hasReviewField).filter(entry -> !MergeReviewIntoCommentMigration.hasCommentField(entry)).forEach(entry -> migrate(entry, parserResult));\n    }\n",
        "\n\n    @Override\n    public void performAction(BasePanel basePanel, ParserResult parserResult) {\n        MergeReviewIntoCommentMigration migration = new MergeReviewIntoCommentMigration();\n        migration.performMigration(parserResult);\n        List<BibEntry> conflicts = MergeReviewIntoCommentMigration.collectConflicts(parserResult);\n        if (!conflicts.isEmpty() && new MergeReviewIntoCommentConfirmationDialog(basePanel.frame().getDialogService()).askUserForMerge(conflicts)) {\n            migration.performConflictingMigration(parserResult);\n        }\n    }\n"
    ]
}

查找失败返回示例:
no sample code
```

## 获取类的基础结构信息接口

### 接口详情

| 地址     | http://106.14.239.166/contest/api/api_structure/ |
| -------- | ---------------------------------------------- |
| 请求方式 | POST                                           |

### 请求参数

| 参数             | 说明           |
| ---------------- | -------------- |
| `qualified_name` | 类的全限定名称 |

### 返回参数

| 返回参数      | 说明                              |
| ------------- | --------------------------------- |
| `extends` | 这个类的父类信息 |
| `fields` | 这个类有哪些Field |
| `fields内部` | 里面是个数组，目前有用的内容在properties中，比如type、qualified_name、full_declaration |
| `implements` |这个类implements哪些，和上面的extends内容差不多，区别在于它是个数组，会有多个|
| `methods` |这个类下面的全部方法，包含了构造函数，里面的doc_info字段comment是用深度学习获取的注释信息，name是方法名|
| `return_value` |每个方法的返回值目前主要有用的内容是qualified_name、type|
| `parameters` |每个方法的参数列表，和返回值不同，它可能会有多个parameter，目前主要有用的内容是qualified_name、type|
